r/RugbyUnion Player Ratings Analysis¶

Introduction¶

Data has been taken from the crowd sourced player ratings from each of the games for the British and Irish Lions. Full credit to the subreddit r/rugbyunion and u/beneath_the_bridge in particular for setting up these ratings. Each player in the squad is rated for their performance (assuming they played) after each match.

Currently data is not yet full as not all matches have been played. However the analysis can be put in place now.

The repo for this project can be found here

Step 2:¶

"Total Rating" is useful but a better measure would be an average as this would account for the number of games played. Otherwise a player who only played a few times, for example Stuard Hogg, would be handicapped by only playing a few games.

In order to calculate a mean it is necessary to have a value of the games played. This means testing each column in the "Games" table and if the value is greater than 0 it increments the "Games_Played" value by one.

This cell throws an annoying exception that is automatically generated, however having looked into it I can't see that it's relevant to the code - I shall ignore for now.

In [42]:
# A total games played column will count the number of non-zero player ratings for each player and store in a new column.
df['Games_Played'] = 0

for i in df['Index']:
    for j in games:
        if df[j][i] != 0.0:
            df['Games_Played'][i] += 1
        else:
            continue

Step 4:¶

Now we can do some actual data analysis using graphs. First I wanted to see if the number of caps that a player had earned for his home nation made any difference to the average rating per game.

With just 4 data points so far this was unlikely to be too reliable as a result but as an issue of personal interest this represented a useful starting point.

As the graph shows its a fairly broad scatter that is unlikely to reveal a serious trend.

Step 5:¶

To test if this graph showed a trend the next step was to create a linear model and run a linRegress. This would show conclusively if any relation existed between the independent and explanatory variable.

It can be seen both from the graph and the linRegress result that no trend exists.

Linregress Output¶

The output of a linregress function call is a list of 5 values.

  • Slope
  • Intercept
  • R-value
  • P-value
  • Standard Error

So it can be seen that the number of caps has very little relevance to the average player rating and no correlation is a fair assumption.

A facile interpretation however suggests that a player with no caps would have an average rating of 6.3 so far this tour.

Step 6:¶

Finally an actually useful graph. This graph simply produces the average team rating (as calculated by taking an average of each players rating) for each game and then plotting them as a bar graph.

The most interesting aspect of this is that, from a brief reading of the data it might be expected that a winning game for the Lions would result in higher ratings for the players, while this has happened to an extent it is not nearly as pronounced as might be expected. Obviously this is likely to change as the data gets filled in.

Step 8:¶

The final graph produced replicates the table above and produces a bar graph of each player's average rating so far. Without published standard error per point these data must be treated with caution, however with just 4 data points the addition of standard error would likely not be clarifying but rather the opposite.

Overall the key things to look at are the best and worst performers according to these ratings. Conor Murray is a clear outlier of the data, this is likely because he had a stand out game and has only played once. Nowell, Henderson and Williams have clearly struggled this tour and are rated less than 5 on average so far.
